Solution Architecture & Lifecycle

How a claim flows end-to-end, and how each component connects: Ingest → Index → Reason → Route → Evaluate → Improve. The reference build runs on the Microsoft stack; this live prototype substitutes Anthropic + Vercel to demonstrate the same lifecycle. The Catastrophic Claims agent already exercises every stage below.

Ingest

Index

Azure AI Search · hybrid

Builds a hybrid index (keyword + vector) over Claims Notes and the catastrophic-injury Guidelines, so the right context can be retrieved.

In this demo: the Retrieve · RAG step surfaces the matching guideline sections.
3

Reason

Azure OpenAI · RAG

Retrieval-augmented generation pulls the right guideline context and classifies whether it's a "seven deadly sins" case — with a confidence score.

In this demo: Claude classifies with evidence, categories and confidence.
4

Route

Confidence-based · Power Apps

High confidence → auto-flag to the catastrophic register. Medium → human review queue. Low → continue standard flow.

In this demo: live routing appears on every verdict.
5

Evaluate

Azure AI Foundry

Runs evals against a golden set, tracking precision and recall so accuracy is measured, not assumed.

In this demo: the Evaluation panel scores Precision / Recall / F1 vs the golden set.
6

Improve

Human-in-the-loop feedback

Human overrides feed back to tune retrieval and prompts; the golden set grows and the model is re-evaluated and redeployed.

In this demo: Agree / Override on a verdict captures feedback for tuning.
